Report¶

Experiment design¶

To discover the difference among the five algorithms and find the best one based on the statistic result, I choose to train the new model several times and using the same training data with a different random seed. And I collect the test score from each run and use autorank to detect the difference among the five groups of test scores.

Autorank will perform a test to reject the null hypothesis that it is no difference among those scores. If the result shows a difference between the five algorithms' values, I can choose the best algorithm by comparing the mean accuracy. The algorithm I choose for task 2 is Gradient boosting. And the design remains the same because to get each score. I use a different random seed to train a new model using the same training set so that this design can compare all the five algorithms.

Reason for the design¶

The hyperparameter for the ensemble algorithms is the number of trees which is 10. The independent variables are algorithm types and datasets, and the dependent variable is the test score of each algorithm. There are ten runs of each model, and each run will create a new model with a different random seed. Then autorank will guarantee it uses an appropriate test and if it rejects the null hypothesis, we can use mean to decide which is the best algorithm.

How the dataset affects the performance of algorithms.¶

First, in the arrhythmia dataset, the algorithm that works best is bagging. This dataset has most of its data which is labeled class one. But the other class has a minimal amount of data, which makes the bagging classifier great because bagging reduces the variance of the results, over-fitting. A simple decision tree will not run well because it can not get enough train on the minority data and can not balance bias and variance. Random forest can be good at unbalanced data, and it is the second best, after Bagging.

In the Caesarian dataset, the best classifier is Adaboosting. There are several possible reasons. First of all, this dataset is tiny, with only 80 samples, which is inappropriate for random forest and bagging, requiring a large amount of data. Secondly, we can see that each class's feature is not entirely prominent from the boxplot, and Adaboost is sensitive to slight differences.

In the website-phishing dataset, the Bagging and Random Forest runs best because the number of features is significant, and there are also sufficient samples on both classes. As a result, it is not very suitable for a simple decision tree. Adaboost is more suitable for features that are not easy to distinguish.

Finally, the dataset that I found best for a simple decision tree has a tiny number of features, nine and two classes. So if we choose a good depth of the tree, it is better than others because others may be overfitting on the training data. And the data has many outliers, which is terrible for Adaboost.

Algorithm performs well on most datasets¶

The algorithm that works best is Random Forest. It performs best in 2 out of 3 datasets, second-best on the other one. It is excellent on data with lots of features. It can handle a large number of input variables. For unbalanced classification data sets, it can balance errors. For datasets with a small number of samples, Random Forest also has good performance because it can assess the importance of features while determining categories.

def readAndModify(path):
    df = pd.read_csv(path)
    for col in df.columns:
        # calculate the number of rows that has "?"
        rowsOfNan = df[col][df[col] == "?"]
        numOfNaN = rowsOfNan.count()
        if(numOfNaN):
            if numOfNaN / len(df.index) > 0.7:
                df.drop(col, axis = "columns",inplace = True)
            else:
                df.drop(rowsOfNan.index, inplace = True)
    return df
